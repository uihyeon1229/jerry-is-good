# 4. 4인 역할 분담 & 1박 2일 타임라인

## 4.1 역할 정의

### 🧑‍💻 A. 인프라/서빙 담당
**핵심**: GPU 인스턴스와 모델 서빙, 그리고 SFT 학습을 책임진다.

**필요 역량**:
- 리눅스 CLI, SSH, tmux/screen
- Python venv, pip 패키지 관리
- CUDA/GPU 기본 이해 (nvidia-smi 읽기)
- HuggingFace transformers/peft 경험 있으면 금상첨화

**Day1**:
- Brev H100 인스턴스 프로비저닝
- Python 가상환경 + vLLM 설치
- Nemotron 3 Nano FP8 다운로드 + vLLM 실행
- API 동작 검증 (curl 테스트)
- Qwen2.5-1.5B 다운로드 (SFT 대기)

**Day1 밤 ~ Day2 새벽**:
- SFT 학습 스크립트 실행 (백그라운드)
- 주기적 loss 확인, 이상 시 개입
- Nsight/DCGM GPU 프로파일 캡처

**Day2**:
- SFT 모델을 별도 포트로 서빙 (비교용)
- 벤치마크 실행 지원
- 발표 시 "인프라 스크린샷" 준비

---

### 🧑‍🔬 B. 파이프라인 담당
**핵심**: NeMo Data Designer + Curator + Guardrails를 엮어 파이프라인을 구성한다.

**필요 역량**:
- Python (중급 이상)
- Pydantic, Jinja2 기본
- YAML 설정 작성
- ML 파이프라인 경험 있으면 유리

**Day1**:
- Data Designer 설치 + 가이드 노트북 완주 (baseline 확보)
- 세법 도메인 스키마 작성 ([03-schema.md](./03-schema.md) 기반)
- Nemotron API 연결 + Sampler/LLMText 첫 5건 미리보기
- Curator 설정 파일 작성
- Guardrails 설정 파일 작성

**Day1 오후**:
- 전체 파이프라인 통합
- 500~1000건 본 생성 트리거 (백그라운드)

**Day2**:
- 필터링 결과 분석 (통과율, 실패 사례)
- 프롬프트 튜닝 (필요 시 2차 생성)
- 최종 JSONL/Parquet 내보내기

---

### 📚 C. 도메인/시드 담당 (세무 실무자)
**핵심**: 세법 정확성, 시드 데이터 품질, 최종 큐레이션을 책임진다.

**필요 역량**:
- **세법 실무 경험** (필수)
- 기본 Python (스크립트 실행 수준)
- MCP/API 호출 기본 이해

**대회 전**:
- 법제처 OC 키 발급 (선임)
- MCP 로컬 테스트
- 대상 세목별 주요 조문 10개씩 목록 작성

**Day1 오전**:
- MCP 시드 수집 스크립트 실행
- 수집된 시드 검수 (조문 누락/중복 확인)
- Judge 루브릭 세부 문구 검토
- 시드 예제 CoT 3~5개 수작업 작성 (Judge Few-shot용)

**Day1 오후~저녁**:
- 생성물 품질 실시간 검수
- "이건 법적으로 틀렸다" 사례 수집 (발표용 안티 패턴)

**Day2**:
- 발표용 하이라이트 10건 큐레이션
- "실무자가 검증한 데이터" 신뢰성 스토리 준비

---

### 🎤 D. 평가/발표 담당
**핵심**: 벤치마크 구성과 발표 스토리텔링을 책임진다.

**필요 역량**:
- 세법 기본 이해 (실무자 아니어도 OK)
- 프레젠테이션 (PPT/Keynote)
- 데이터 시각화 (matplotlib/간단 그래프)
- 영어 발표 가능하면 가점 (NVIDIA 심사위원 대응)

**대회 전**:
- 발표 템플릿 준비 (NVIDIA 스폰서 로고 포함)
- 벤치마크 20문제 초안 작성 (C 담당 리뷰)

**Day1**:
- 벤치마크 세트 최종화 (세목별 5문제 × 4세목)
- MCP로 정답·근거조문 사전 확보
- 발표 첫 5슬라이드 초안

**Day1 저녁**:
- Before(Base 모델) 답변 수집 및 채점

**Day2 오전**:
- After(SFT 모델) 답변 수집 및 채점
- Before/After 차트 제작
- 발표자료 완성 + 데모 동영상

**Day2 오후**:
- 리허설 2회
- Q&A 예상 질문 준비
